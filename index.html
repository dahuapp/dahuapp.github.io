Summary

Dahu is an alternative to existing screencasting software. It allows combining a set of screenshots into an interactive presentation.

Dahu's principle

Dahu is based on few ideas that distinguish it from other existing screencasting software:

  • Take screenshots as still images, not video. Videos are hard to edit and hard to navigate. Badly compressed videos are hardly readable. Instead, Dahu uses a succession of still images, that look like a video to the user. This approach is highly inspired from Wink.
  • Capture mouse position. While capturing, the mouse position for each step of the animation is captured. The generated animations show a smooth mouse movement from the starting point to the end point. Mouse positions can be edited after capture.
  • Annotate with tooltips, not voice. The explanations on the presentation appear as textual tooltips. This makes post-editing easy. The text is rendered as normal text in the presentation webpage, hence can be copy-pasted, searched, and indexed by search engines.
  • Standard web technologies. No flash, no plugin. The generated animation use only standard web technologies (HTML, CSS, JavaScript). The can be viewed on any modern browser, without any plugin (not even flash). Among other advantages, this is what allows search engines to index presentation, and makes it easy to use any HTML elements like links within tooltips.
  • Multi-platform. The tool itself is written in Java and JavaScript. It requires a recent version of Java, but runs on Windows, Linux, Mac OS, and probably other platforms as well.
  • Free and Open-Source. Dahu is available for free (both free of charge and open-source).
  • Version-Control Friendly. A Dahu project consists in one small text file (JSon) presentation.dahu, plus one image file per screenshot, named after a UUID. As a result, using a version control system (like Git, Mercurial, SVN...) is reasonably efficient (a change to a single image or to presentation.dahu does not require storing a full version of the whole project). With a bit of care, branching and automatic merging are possible.

Dahu's Development Status

Dahu is still in early stage of development. The presentation generation works well, but not all features are implemented in the graphical editor. To display tooltips in presentations, or use any advanced features, you will need to use a text editor and edit the presentation.dahu file (the user-manual explains how to do this).

In other words, creating a presentation requires some computer-science skills. The generated presentations are already user-friendly however. They require a modern web browser (Chromium, Firefox, IE version 8 or greater, ...).

Download

The source code can be downloaded from GitHub. You will need Java >= 1.7 and Gradle to compile it.

Other Screencasting Software

Even if there is a wide variety of software that offers to make screencasts, none of these programs do offer effective solutions combining the features / properties that we could expect of this kind of tool.

Generally the approach is to make a video recording of an area of your screen and to couple the audio. This approach sets several problems for the capture (an error in your manipulation or speech implies restarting everything or do some video collage), post-editing (adding text, animations, and other, means video editing), export (the formats are owner, and often too heavy), indexing (very difficult to index a video or a Flash file), reading (not practical to have to move the cursor to find the piece of information we are looking for: the time or this or that thing is explained), aso..

A notable exception is Wink, which was a great source of inspiration for Dahu, but it is not open-source, runs very badly on Linux, and does not seem to be maintained anymore. Also, Wink generates Flash animations, which was a sensible choice a few years ago, but is no longer a good idea: browser already offer the required features without a plugin, and Flash is not available on all platforms (e.g. iPad).

Other alternatives include Adobe Captivate (commercial, not open-source) and salasaga (does not seem maintained).

Architecture

Dahu consists in three parts:

  • Capture: a tool for capturing and editing, written in JavaFX and Java. It will allow : a) the recording of information linked to the screencasts and b) post-editing screencasts to add meta-data (eg, texts, animations and information about screencasts.
  • Intermediate Format: an intermediate format to store the content, form and dynamics of screencasts. It is a simple JSon-based format, easily extensible and hand-editable if needed.
  • Reading: a viewer to view screencasts in a web browser (HTML5, JS, CSS).
